\title{\textbf{Simulation Debugging Report}\\
\large Modifications Attempted on the MNRS Quantum Walk Search Simulation}
\author{Jacopo Bellosi}

\noindent
This report documents all main modifications attempted on the simulation code for the paper
\textit{``Solving the Subset Sum Problem via Quantum Walk Search''} (Lancellotti et al., IEEE TC 2025).
All tests use the same instance: $n=3$, $k=1$, \texttt{values=[1,2,3]}, \texttt{target\_sum=3}.
In every case the final result is a flat probability distribution: \texttt{0.333 |1>, 0.333 |2>, 0.333 |3>}.

% -----------------------------------------------------------------------
\section*{Baseline -- Original Paper Code}
% -----------------------------------------------------------------------

Code as provided with the paper (\texttt{code\_2025TC/cssp.py}).
Key settings: $\delta = n/(k(n-k))$ (unclamped),
$\ell_s = \lceil\log_2(\pi/(2\sqrt{\delta}))\rceil$,
outer iterations $= \lceil\sqrt{\binom{n}{k}}\rceil$,
forward QFT, single $U_U$ application per QPE qubit.

\begin{lstlisting}[language=Python]
delta = n / (k * (n - k))
len_s = int(np.ceil(np.log2(np.pi / (2 * np.sqrt(delta)))))
n_external_iters = int(np.ceil(np.sqrt(comb(n, k))))

for qw_iter in range(len_s):
    prw.apply(qrw_update.dag(), ...)
    for j in range(k):
        prw.apply(X, wstate_ones[j])
    prw.apply(Z.ctrl(k), qpe_s[qw_iter], wstate_ones)   # only wstate_ones
    for j in range(k):
        prw.apply(X, wstate_ones[j])
    prw.apply(qrw_update, ...)
    # Ref B controlled only on wstate_zeros
    prw.apply(Z.ctrl(n - k), qpe_s[qw_iter], wstate_zeros)

prw.apply(QFT(len_s), qpe_s)   # forward QFT
\end{lstlisting}

\section*{Modification 1 -- Fix Reflection Logic}
% -----------------------------------------------------------------------

Reflection B was corrected: added the update of \texttt{wstate\_zeros}
(the $\omega'$ register) symmetrically with \texttt{wstate\_ones}, and
added the corresponding X-gate flips before and after the controlled-Z.

\begin{lstlisting}[language=Python]
# Before: only controlled on wstate_ones (size k)
prw.apply(Z.ctrl(k), qpe_s[qw_iter], wstate_ones)

# After: Ref A controlled on wstate_ones, Ref B controlled on wstate_zeros
# Ref A
for j in range(k):
    prw.apply(X, wstate_ones[j])
prw.apply(Z.ctrl(k), qpe_s[qw_iter], wstate_ones)
for j in range(k):
    prw.apply(X, wstate_ones[j])
prw.apply(qrw_update, ...)
# Ref B
prw.apply(qrw_update.dag(), node_s_zeros, node_s_ones, ...)
for j in range(n - k):
    prw.apply(X, wstate_zeros[j])
prw.apply(Z.ctrl(n - k), qpe_s[qw_iter], wstate_zeros)
for j in range(n - k):
    prw.apply(X, wstate_zeros[j])
prw.apply(qrw_update, node_s_zeros, node_s_ones, ...)
\end{lstlisting}

\section*{Modification 4 -- Spectral Gap Clamping and Minimum QPE Qubits}
% -----------------------------------------------------------------------

For $n=3$, $k=1$, the formula gives $\delta = 1.5 > 1$, which is
physically invalid. Also, $\ell_s$ computed from unclamped $\delta$
evaluates to 0 (i.e., 1 qubit QPE), insufficient for Szegedy coherence.

\begin{lstlisting}[language=Python]
# Before:
delta = n / (k * (n - k))
len_s = int(np.ceil(np.log2(np.pi / (2 * np.sqrt(delta)))))

# After:
delta = min(n / (k * (n - k)), 1.0)
len_s = max(int(np.ceil(np.log2(np.pi / (2 * np.sqrt(delta))))), 3)
\end{lstlisting}

\noindent\textbf{Result} (\texttt{test\_finale4.txt} -- same flat output):
\begin{lstlisting}[language={}]
DEBUG: Search space N=3, performing 1 Grover iterations
{'nbqbits': 28, 'size': 147, 'gate_size': 1366}
0.33333333333333315 |1>
0.3333333333333331  |2>
0.33333333333333326 |3>
\end{lstlisting}

\section*{Modification 5 -- Correct Grover Iteration Count}
% -----------------------------------------------------------------------

The outer loop count was changed from $\lceil\sqrt{N}\rceil$ (which gives 2
for $N=3$, causing over-rotation) to the optimal Grover count
$\lfloor\frac{\pi}{4}\sqrt{N}\rfloor$ (which gives 1 for $N=3$).

\begin{lstlisting}[language=Python]
# Before:
n_external_iters = int(np.ceil(np.sqrt(comb(n, k))))

# After:
N_search_space = comb(n, k)
n_external_iters = int(np.floor((np.pi / 4) * np.sqrt(N_search_space)))
if n_external_iters == 0 and N_search_space > 1:
    n_external_iters = 1
\end{lstlisting}

\noindent\textbf{Result}: flat, same as above.

% -----------------------------------------------------------------------
\section*{Modification 6 -- QPE Walk Loop Exponential Scaling}
% -----------------------------------------------------------------------

Standard QPE requires the walk operator $W$ to be applied $2^j$ times for
the $j$-th QPE qubit. The original code applied it once per qubit.

\begin{lstlisting}[language=Python]
# Before (one application per QPE qubit):
for qw_iter in range(len_s):
    prw.apply(qrw_update.dag(), ...)
    # ... reflections ...
    prw.apply(qrw_update, ...)

# After (2^j applications for qubit j):
for qw_iter in range(len_s):
    for _ in range(2**qw_iter):
        prw.apply(qrw_update.dag(), ...)
        # ... reflections ...
        prw.apply(qrw_update, ...)
\end{lstlisting}

\noindent\textbf{Result}: flat, same as above.

\section*{Modification 8 -- Dicke Register Uncomputation}
% -----------------------------------------------------------------------

After VBE, the $\sigma$ register remains entangled with $S$, acting as a
which-path observer. A dedicated routine was added to uncompute $\sigma$
to $|0\rangle^n$ using the content of $S$.

\begin{lstlisting}[language=Python]
# After Dicke + VBE:
prw.apply(generate(n, k), dicke)
prw.apply(bix.bix_matrix_compile_time(n, 1, m, k, sorted_values),
          dicke, node_s_ones, node_s_zeros)

# NEW: uncompute dicke register
inv_dicke_rout = build_uncompute_dicke_from_values(n, k, m, sorted_values)
flat_s_ones = [q for reg in node_s_ones for q in list(reg)]
prw.apply(inv_dicke_rout, flat_s_ones, dicke)
\end{lstlisting}

\noindent\textbf{Result}: flat, same as above.

\section*{Modification 9 -- Reflection B: Swap S$\leftrightarrow$T Registers}
% -----------------------------------------------------------------------

Reflection B should project from the perspective of the \emph{adjacent}
vertex, so $U_U$ must be applied with $T$ acting as source (swapped with
$S$). Also changed the controlled-Z to act jointly on both $\omega$ and
$\omega'$ (full $n$-qubit control).

\begin{lstlisting}[language=Python]
# Before (Ref B with S' as source, controlled on wstate_zeros only):
prw.apply(qrw_update.dag(),
          node_s_zeros, node_s_ones, node_t_zeros, node_t_ones,
          alpha_zeros, alpha_ones, wstate_zeros, wstate_ones)
prw.apply(Z.ctrl(n - k), qpe_s[qw_iter], wstate_zeros)
prw.apply(qrw_update,
          node_s_zeros, node_s_ones, ...)

# After (Ref B with T as source, joint control on wstate_ones + wstate_zeros):
prw.apply(qrw_update.dag(),
          node_t_ones, node_t_zeros, node_s_ones, node_s_zeros,
          alpha_ones, alpha_zeros, wstate_ones, wstate_zeros)
for j in range(k):
    prw.apply(X, wstate_ones[j])
for j in range(n - k):
    prw.apply(X, wstate_zeros[j])
prw.apply(Z.ctrl(n), qpe_s[qw_iter], wstate_ones, wstate_zeros)
for j in range(k):
    prw.apply(X, wstate_ones[j])
for j in range(n - k):
    prw.apply(X, wstate_zeros[j])
prw.apply(qrw_update,
          node_t_ones, node_t_zeros, node_s_ones, node_s_zeros, ...)
\end{lstlisting}

\section*{Root-Cause Diagnosis}
% -----------------------------------------------------------------------

After ten independent modifications, all producing the same
$(\tfrac13,\tfrac13,\tfrac13)$ output, the failure mode is not a
single localized bug but a mismatch between the test instance and the
regime in which the paper's MNRS construction is well-defined.

\paragraph{1. Spectral-gap violation.}
The paper assumes $\delta = n/(k(n-k))$ as the spectral gap of $J(n,k)$,
which is valid only when the resulting value is $\le 1$. For $J(3,1)$ we
obtain
\[
\delta = \frac{3}{1\cdot 2} = 1.5,
\]
which is mathematically impossible for a spectral gap
($|\lambda_2| < 1 \Rightarrow \delta \in (0,1]$). The formula degenerates
because for $J(n,1) = K_n$ (complete graph) all non-Perron eigenvalues are
$-1/(n-1)$, giving true $\delta = n/(n-1) \cdot (1 - 1/(n-1))$ only for
$n\ge 3$; the paper's closed form is an asymptotic expression that is
meaningful for $k(n-k) \ge n$, i.e. for $k\ge 2$ and $n\ge 4$. $J(3,1)$
and $J(4,2)$ (where $\delta=1$ exactly) are degenerate boundary cases.

\paragraph{2. QPE register too narrow.}
From $\delta=1.5$ the code computes
\[
\ell_s = \left\lceil \log_2\!\frac{\pi}{2\sqrt{\delta}} \right\rceil
       = \lceil 0.36 \rceil = 1.
\]
With a single QPE qubit the approximate reflection
$U_{\text{ref}}(E) \approx \mathrm{QFT}^\dagger \cdot (I - 2|0\rangle\!\langle 0|) \cdot \mathrm{QFT}$
cannot separate the phase-$0$ eigenspace (the stationary edge $|E\rangle$)
from phases $\theta \in (0,\pi)$ with the resolution required by the MNRS
analysis. The ``reflection'' becomes a noisy phase scrambler:
non-stationary eigenvectors are not mapped back to $|+\rangle^{\otimes s}$
on the QPE register, leaving ancilla entanglement that erases any
amplitude amplification when the QPE register is traced out.

\paragraph{3. Outer-loop count too small to compensate.}
$\lceil\sqrt{\binom{3}{1}}\rceil = 2$ (and the Grover-optimal
$\lfloor\tfrac{\pi}{4}\sqrt{N}\rfloor = 1$) gives too few iterations to
re-concentrate amplitude when the inner reflection is this imprecise.

\paragraph{4. Why it runs for hours.}
The circuit uses $28$ qubits (Dicke $3$ + $S_1/S_0/T_1/T_0$ each $2$--$4$
+ $\alpha,\omega$ registers + QPE $1$ + sum $2$). \texttt{PyLinalg} is a
full state-vector simulator, so each of the $\sim 2500$ gates touches a
vector of $2^{28}\approx 2.7\cdot 10^8$ complex amplitudes on a single
CPU core. Runtime is dominated by the $\sim$1600 CNOTs in the VBE/arith
routines, not by the walk operator itself. \emph{This is a cost of
PyLinalg, not a code defect.}

\paragraph{5. The role of \texttt{qatext}/\texttt{qat-utils}.}
The missing or present state of \texttt{qatext} affects compile-time
plumbing (Dicke preparation, BIX data structure, arithmetic decomposition)
but does \emph{not} change the algorithmic behavior. All sub-routines
execute as expected; the flat distribution is produced by the algorithm
itself, not by a missing decomposition.

% -----------------------------------------------------------------------
\section*{Verdict -- Is \texttt{cssp.py} Repairable for $J(3,1)$ and $J(4,2)$?}
% -----------------------------------------------------------------------

\textbf{No, not within the MNRS framework as presented in the paper.}

\begin{itemize}
\item For $J(3,1)$: the instance violates $\delta\le 1$. No choice of
$\ell_s$, QFT direction, walk-power scaling, or reflection-B variant will
produce a non-uniform distribution on $S_1$, because the approximate
reflection cannot act as $-I$ on $|E\rangle$ with $\le 1$ QPE qubit, and
increasing $\ell_s$ artificially cannot create a spectral gap that the
graph does not have.

\item For $J(4,2)$ ($\delta=1$ exactly): the algorithm is on the
boundary. Increasing $\ell_s$ to $3$--$5$ and using inverse QFT with
$2^j$ walk-power scaling gives a marginal amplification, but with only
$N = \binom{4}{2} = 6$ vertices a direct Grover search is both simpler
and strictly better.

\item For $J(n,k)$ with $\delta < 1$ and $n\ge 5$, $k\ge 2$: the corrected
MNRS circuit (inverse QFT, exponential walk scaling, symmetric reflection
B, $\ell_s\ge 3$) \emph{should} amplify, but the qubit count
($\Theta(n\log n)$) puts any such instance out of reach of PyLinalg on a
laptop.
\end{itemize}

\paragraph{Practical recommendation.}
For the demonstration instances $J(3,1)$ and $J(4,2)$ the honest
working implementation is plain Grover on the vertex register: it uses
$O(\sqrt{N})$ oracle calls, $\log_2 N$ qubits for the search space,
and reaches $>90\%$ success probability in one iteration. The
MNRS/Szegedy walk described in the paper is asymptotically superior
but its advantage only materializes for instances too large to simulate
classically. This is a limitation of the simulation target, not of the
paper's theoretical construction.
