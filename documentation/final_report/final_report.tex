\section{Introduction}
\label{sec:intro}
% =====================================================================

\subsection{Goal of the project}

The goal of this Software Engineering 2 research project was to
\emph{reproduce the results} of the paper \emph{``Solving the
Subset Sum Problem via Quantum Walk Search''} (Lancellotti,
Perriello, Barenghi, Pelosi; IEEE Transactions on Computers,
vol.~75, n.~1, 2026). The paper proposes a gate-level quantum
algorithm for the Constrained Subset Sum Problem (CSSP) and
publishes a reference implementation in \texttt{myQLM}; our task
was to read this material, run it on a small instance, and check
whether the simulated output matched the expected behaviour.

\subsection{The paper in one paragraph}

The Subset Sum Problem (SSP) asks whether a subset of a given
multiset of integers sums to a target value $p$. The
\emph{constrained} variant (CSSP) further requires the subset to
contain exactly $k$ elements. SSP is NP-complete and underlies
several post-quantum cryptographic constructions, so it is a
natural target for quantum algorithms. The paper attacks CSSP with
a Magniez–Nayak–Roland–Santha (MNRS) quantum walk on the Johnson
graph $J(n,k)$, whose vertices are the candidate $k$-subsets and
whose edges connect subsets that differ in a single element. The
walk is interleaved with a sum-checking oracle and with a
QPE-based approximate reflection so that, after a number of
iterations proportional to $\sqrt{\binom{n}{k}}$, the marked
solution is amplified. The expected speed-up over a brute-force
classical search is therefore quadratic, which is the same gain
that Grover's algorithm provides in the unstructured case.

\subsection{Activities carried out}

The work was organised in nine steps, all reflected in the
repository:
\begin{enumerate}[leftmargin=1.4em]
\item Studied the basics of quantum computing and of the
\texttt{Qiskit} framework, used as a reference language for
quantum circuits.
\item Read the paper in depth, extracting notes on the algorithm
and on the role of each sub-routine.
\item Studied the reference implementation
(\texttt{paper-codes/2025-TC}, mirrored locally in
\texttt{code\_2025TC/}) and reconstructed the high-level structure
of \texttt{cssp.py}.
\item Executed the code on the smallest instance compatible with
classical state-vector simulation, $J(3,1)$.
\item Attempted local modifications to fix the unexpected output.
\item Produced a structured comparison between the paper and the
implementation, in the file
\texttt{documentation/Code and test mapping.pptx}, to check
whether the code met the expected functional standards.
\item Read a set of related works
(\texttt{documentation/paper\_comparison/}) and compared their
methods with ours, identifying the main differences.
\item Identified the main problems in the code together with
their underlying reasons.
\item Submitted the agreed fixes to the upstream repository as a
pull request.
\end{enumerate}

The remainder of the report follows the same logical order:
Section~\ref{sec:paper} describes the paper and the original
behaviour of the code; Section~\ref{sec:work} reports the
modifications we applied; Section~\ref{sec:compare} discusses how
related works compare with ours; Section~\ref{sec:mapping}
presents the analysis based on the code-and-test mapping;
Section~\ref{sec:concl} draws the conclusions.

\section{Brief explanation of the paper and of the original code}
\label{sec:paper}
% =====================================================================

\subsection{What the paper does}

The paper builds an end-to-end quantum circuit for CSSP from four
ingredients.

\paragraph{(1) Johnson graph $J(n,k)$.} Each vertex is a $k$-subset
of $\{1,\dots,n\}$; two vertices are adjacent if and only if they
share $k-1$ elements. The graph has $\binom{n}{k}$ vertices and
the closed-form spectral gap
\[
\delta = \frac{n}{k(n-k)},
\qquad \text{valid for } k(n-k)\ge n.
\]

\paragraph{(2) Marking oracle $U_f$.} Quantises the indicator
function of the solution: it computes the sum of the elements of
the current subset in an ancillary register and flips the global
phase whenever the sum equals the target.

\paragraph{(3) Walk operator $U_W$.} The Szegedy quantisation of
one classical step of the walk on $J(n,k)$. It is a double
reflection, $U_W = U_{\mathrm{ref}}(a_{v_j})\,U_{\mathrm{ref}}(a_{v_i})$,
in which the basic ``update'' sub-routine $U_U$ generates the
superposition of the neighbours of the current vertex.

\paragraph{(4) Approximate reflection via QPE.} The amplitude
amplification step needs a reflection $U_{\mathrm{ref}}(E)$
around the stationary state $|E\rangle$ of the walk. Since
$|E\rangle$ has eigenphase $0$ under $U_W$ and the other
eigenstates have non-zero eigenphase, a Quantum Phase Estimation
(QPE) on $U_W$ acts as an approximate reflection that flips the
sign of every state whose estimated phase is non-zero. The MNRS
precision requirement fixes the number of QPE qubits to
$\ell_s = \lceil \log_2(\pi/(2\sqrt\delta)) \rceil$.

What the paper \emph{does not} do is run the algorithm
end-to-end. The authors validate the building blocks separately
and explicitly state that an end-to-end execution is not
feasible: the cryptographic regime they target is $n\ge 25$ for
the T-count advantage and $n\ge 2^{13}$ for the T-depth
advantage, well beyond any classical state-vector simulator.

\subsection{Initial doubts: the flat distribution}

Our first attempt was to run \texttt{cssp.py} as it is, on the
toy instance $n=3$, $k=1$, $\mathtt{values}=[1,2,3]$,
$\mathtt{target\_sum}=3$. With $N=\binom{3}{1}=3$ admissible
subsets and the unique marked solution $S=\{3\}$, an amplifying
algorithm should produce a distribution close to $(0,0,1)$ on
the vertex register $S_1$. Instead, the simulation returned a
\emph{flat} distribution, identical to the initial Dicke state.
Table~\ref{tab:baseline} summarises the direct run and the run
instrumented with intermediate checkpoints.

The fact that the oracle flipped the right phase but the
distribution remained flat raised three independent suspicions
that we explored in the following weeks: a possible bug in the
walk reflections, a possible bug in the cleanup of the ancillary
registers, and a possible structural mismatch between the
algorithmic regime and the toy instance we had chosen.

\section{What we did to the code}
\label{sec:work}
% =====================================================================

This section is a condensed version of
\texttt{documentation/modification\_report/modification\_report.tex},
which contains the full diff-level chronology. We collected the
work in five modifications, applied between mid-March and
mid-April 2026 and validated against the simulation logs in
\texttt{documentation/quantum\_simulation/} (files 01--13). The
modifications were also discussed with one of the paper authors,
who clarified the intended behaviour of the most ambiguous
sub-routines and confirmed which of our patches were genuine
fixes; we do not report the content of that exchange here.

\subsection{Modification 1 — Symmetric Reflection A and B on the
full coin}

The MNRS walk operator decomposes as a double reflection in which
the inner reflection $R_0 = 2|0\rangle\langle 0|-I$ acts on the
\emph{whole} coin register $\omega = (\omega_1,\omega_0)$ of $n$
qubits. In the original code the two reflections were asymmetric:
Reflection~A controlled the $Z$ flip on $\omega_1$ only ($k$
qubits), and Reflection~B did the analogous thing on $\omega_0$,
with a typo on the loop bound of one of the $X$-sandwiches. The
fix promotes both reflections to a multi-controlled $Z$ on the
full $n$-qubit coin.

\noindent\textbf{Effect.} Gate count rises from $2547$ to $2581$,
qubit count unchanged ($28$). The marginal distribution on the
toy instance is still flat.\\
\noindent\textbf{Verdict.} Genuine bug. Necessary for the
implementation to match the algorithm described in the paper.

\subsection{Modification 2 — Removal of the $\alpha/\omega$
cleanup block}

Inside the QPE block, after the $\ell_s$ controlled walk steps,
the original code performed a manual ``cleanup'' of the ancillary
registers $\alpha$ and $\omega$ by re-applying
\texttt{copy\_register} and \texttt{generate}. At that point in
the circuit those registers are not in $|0\rangle$: they are
entangled with the visited vertices. Re-applying \texttt{generate}
to a non-zero state corrupts the entanglement instead of
restoring it. The correct uncomputation is already provided by
\texttt{prw.uncompute()} at the end of the \texttt{compute()}
block. We therefore removed the manual cleanup.

\noindent\textbf{Effect.} Gate count drops from $2547$ to $1359$
($-47\%$), qubit count unchanged. Distribution still flat.\\
\noindent\textbf{Verdict.} Genuine bug. The cleanup was both
redundant and destructive on instances with a non-trivial
amplification window. A more refined per-element reset would be
preferable but lies outside the scope of our patches.

\subsection{Modification 3 — Use \texttt{insert\_lw} instead of
\texttt{insert\_ld}}

The \texttt{delete} routine in
\texttt{sliding\_sort\_array.py} internally called
\texttt{insert\_ld(...).dag()}. A reversibility check showed that
\texttt{insert\_ld} is not properly reversible: composing
\texttt{delete} with \texttt{insert\_ld} on a small input does
not give the identity, because of garbage left in the ancillas (a
comment in the source already flagged the routine as such). Since
the walk operator chains many \texttt{insert}/\texttt{delete}
calls, the garbage propagates and degrades the QPE step. The fix
routes \texttt{delete} through \texttt{insert\_lw(...).dag()},
which passes the reversibility check.

\noindent\textbf{Effect.} Gate count $1359 \to 1366$ (negligible),
distribution still flat.\\
\noindent\textbf{Verdict.} Structural fix. Without it the walk
operator is not unitary on the ancillary subspace.

\subsection{Modification 4 — Clamp $\delta\le 1$ and force
$\ell_s\ge 3$}

The closed-form gap $\delta = n/(k(n-k))$ is derived under the
hypothesis $k(n-k)\ge n$. On $(n,k)=(3,1)$ it returns $\delta=1.5$,
a numerical artifact (a true spectral gap of a stochastic matrix
lies in $(0,1]$). Plugging this into the precision formula gives
$\ell_s=1$, which makes the QPE register degenerate. We clamp
$\delta$ to $1$ and force $\ell_s$ to a minimum value, so that
the QPE block is at least well-defined on small instances.

\noindent\textbf{Effect.} Qubit count rises from $28$ to $30$;
gate count unchanged; distribution still flat.\\
\noindent\textbf{Verdict.} Defensive patch, not an algorithmic
fix. It cannot create a spectral gap that the graph does not
have. On graphs with $\delta<1$ it never triggers, so it is fully
backward compatible.

\subsection{Modification 5 — Diagnostic checkpoints}

The instrumented copy \texttt{cssp\_with\_checkpoint.py}
simulates the full state at six points of the circuit and prints,
for each state with non-negligible probability, the amplitude and
the value of every named register. It is not an algorithmic
change, but it was decisive: it pinpointed that the oracle is
correct (phase flip at Checkpoint~3) and that the failure sits
inside the QPE-based reflection, between Checkpoints~3 and~5.

\noindent\textbf{Effect.} No change in the circuit, but empirical
confirmation that the bugs in Modifications~1--3 are not the only
reason for the flat output.\\
\noindent\textbf{Verdict.} Indispensable observability tool;
without it, the diagnosis ``the regime is wrong, not only the
code'' would have remained a conjecture.

\subsection{Why their work can be simulated and ours cannot}

Reading the seven papers in parallel, a clear pattern emerges:
each work that successfully validates a quantum algorithm in
simulation does so by sidestepping the precise constraint that
breaks our toy run. We can group the workarounds in three
families.

\paragraph{Family A — replace MNRS with a different algorithm.}
Zheng et al.\ tackle SSP itself, but with QPE followed by
Amplitude Amplification on a direct sum oracle, with no Johnson
graph and no walk. Sato et al.\ adopt a similar philosophy on
TSP. Removing the walk also removes the spectral-gap dependence
and the QPE precision requirement, which makes the small-scale
simulation feasible. The cost is methodological: one loses the
Markov-chain interpretation and the asymptotic complexity becomes
the standard Grover bound, not the walk speed-up.

\paragraph{Family B — keep the walk, but pick easy graphs or
skip the reflection.} Razzoli et al., Portugal \& Moqadam and
Wing-Bocanegra all run quantum walks in simulation, but on
graphs whose spectrum is either trivial ($K_N$, $K_{n,n}$ have
two-eigenvalue spectra), available in closed form (cycles), or
sidestepped by running a single step. None of them uses a
Johnson graph $J(n,k)$ with $k\ge 2$, where the spectrum has
$k+1$ distinct eigenvalues whose ratios depend on $n,k$ in a
non-trivial way. Equally important, none of them uses an MNRS
\emph{approximate reflection} via QPE: they either use no
reflection at all, or implement a direct Grover-style
reflection.

\paragraph{Family C — give up on end-to-end simulation.}
Perriello et al.\ are the most candid: working on Information
Set Decoding at cryptographic sizes, they explicitly state that
end-to-end simulation is impossible and only validate the
sub-components. Their numbers are resource estimates, not
measured probabilities. This is methodologically the same
trajectory taken by Lancellotti et al.: validate the building
blocks separately, present analytical complexity figures, and
renounce the end-to-end output distribution.

\subsection{The key computational difference}

The pieces of the MNRS pipeline that make it powerful at
cryptographic scale --- the Johnson graph encoding, the
approximate reflection via QPE, the spectral-gap dependence ---
are precisely the pieces that make it impossible to validate at
toy scale. The other works either remove one of these pieces or
operate at a scale at which the simulator can still keep up.
The combination ``MNRS + Johnson graph + small instance +
classical simulation + amplitude check'' is, to the best of our
literature survey, an empty intersection. Our finding that the
toy run on $J(3,1)$ produces a flat distribution is therefore
consistent with the literature, not anomalous.

\section{Analysis based on the code-and-test mapping}
\label{sec:mapping}
% =====================================================================

\subsection{What the mapping was for}

In parallel with the modifications, we built a structured
comparison between the paper and the implementation, stored as
\texttt{documentation/Code and test mapping.pptx}. The mapping
goes through every algorithmic block described in the paper
(Dicke preparation, vertex binary encoding, update operator,
oracle, QPE-based reflection, diffusion, output measurement) and
locates the corresponding sub-routine and tests in the code. For
each block we then asked three questions:
\begin{enumerate}[leftmargin=1.4em]
\item Does the implementation behave as the paper describes, on
inputs we can simulate?
\item Are the tests in the repository sufficient to validate this
behaviour?
\item Does the documentation make it easy for a new reader to
match the code with the paper?
\end{enumerate}

The mapping turned out to be the most useful artefact for the
project: it organised the discussion with the corresponding
author and produced the short list of items that we then took to
the upstream pull request.

\subsection{Critical points that we resolved}

The following items came out of the mapping, were agreed with the
corresponding author, and were addressed by the pull request that
we opened against the paper's repository.

\begin{itemize}[leftmargin=1.2em]
\item \textbf{Asymmetric reflections (Modification~1).} Both
reflections inside the QPE block now control the $Z$ flip on the
full coin register, with the correct $X$-sandwich on both halves.
\item \textbf{Degenerate QPE register on small instances
(Modification~4).} The lower bound on $\ell_s$ has been raised so
that the QPE register is never trivial; the change is invisible
on instances where the original formula already returned a
sensible value.
\item \textbf{Documentation gap.} The original \texttt{README}
made it difficult for a new reader to understand the entry point
of the code, the meaning of the command-line flags, and the
assumed installation. We rewrote it with a citation to the
paper, an explicit list of requirements, and a usage block that
matches the actual CLI of \texttt{cssp.py}.
\end{itemize}

\subsection{Critical points that remain open}

A second set of items emerged from the mapping but lies outside
the scope of a small, targeted pull request. We list them here
for completeness, since they would be the natural follow-ups of
this project.

\begin{itemize}[leftmargin=1.2em]
\item \textbf{End-to-end validation is structurally impossible
on a state-vector simulator.} Already $J(4,2)$ requires more than
$30$ qubits and exceeds the practical ceiling of \texttt{PyLinalg}
on commodity hardware. As discussed in Section~\ref{sec:compare},
this is not a code defect but a property of the algorithmic
regime targeted by the paper.
\item \textbf{Per-element reset of the cleanup ancillas.} The
removal of the destructive cleanup block (Modification~2) is a
local mitigation: a proper reset of the $\alpha/\omega$ registers
that does not touch the entangled state would require a different
construction, beyond the scope of the present pull request.
\item \textbf{Test coverage of the composite walk operator.} The
existing tests cover the building blocks (insert, delete, oracle,
Dicke preparation) in isolation, but no test exercises the full
walk operator $U_W$ as a single unit on a small Johnson graph.
The bugs identified in Modifications~1 and~3 would have been
caught earlier by such a test.
\item \textbf{Mapping of paper symbols to code identifiers.} The
register names in the code do not always match the symbols used
in the paper (for instance the role of $\alpha_0$ versus
$\alpha_1$, or of the two halves of the coin). A short
correspondence table in the repository would significantly
shorten the on-boarding time.
\end{itemize}

\section{Conclusions}
\label{sec:concl}
% =====================================================================

\subsection{What was accomplished}

The project produced four concrete outcomes:
\begin{enumerate}[leftmargin=1.4em]
\item A patched local copy of \texttt{cssp.py} in
\texttt{code\_updated/}, with the four fixes and the diagnostic
checkpoints.
\item A modification report
(\texttt{documentation/modification\_report/modification\_report.tex})
that documents the chronology and the reasoning behind every
change.
\item A code-and-test mapping
(\texttt{documentation/Code and test mapping.pptx}) that aligns
each algorithmic block of the paper with the corresponding
sub-routine in the code, and that produced the list of agreed
fixes.
\item A pull request to the upstream repository
\texttt{paper-codes/2025-TC} (PR \#1, branch
\texttt{correctness/mod1-mod4}), containing the symmetric
reflections, the lower bound on $\ell_s$, and the rewritten
\texttt{README}. At the time of writing, the pull request is
under review by the corresponding author and is awaiting merge.
\end{enumerate}

\subsection{Final remarks}

The MNRS pipeline of Lancellotti et al.\ is algorithmically
correct once the bugs identified in Modifications~1--3 are
removed. It cannot, however, be executed in a meaningful way on
the toy instances accessible to a state-vector simulator: on
$J(3,1)$ the spectral gap is degenerate and the algorithm has no
amplification window to exploit; on $J(4,2)$ and beyond the
qubit count exceeds the simulator budget. The paper itself is
explicit about this trade-off and targets cryptographic-scale
instances on fault-tolerant hardware.

The literature comparison confirms that this is a structural
limitation of the family of algorithms, not an oversight of the
authors: no surveyed work successfully simulates an MNRS walk on
a Johnson graph end-to-end, and the working alternatives
correspond to relaxing one of the constitutive ingredients
(switching to direct Grover, choosing a graph with a trivial
spectrum, or limiting validation to sub-components).

For a future continuation of the project, the natural next step
would be to follow Path~A from the modification report --- a
direct Grover search on a compact vertex register --- and use it
as a baseline against which the qualitative behaviour of a
stripped-down MNRS variant can be compared. Until then, the
contribution of this work is the diagnosis of the limits, the
upstream fixes that bring the reference implementation closer to
the paper, and the documentation needed for a new reader to pick
it up where we left off.
